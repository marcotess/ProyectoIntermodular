# Conclusiones y mejoras futuras

## 1. Conclusiones

El proyecto cumple el objetivo principal de centralizar la gestion academica de cursos, proyectos y documentacion asociada en una sola aplicacion web. Frente a un seguimiento disperso, la solucion desarrollada aporta una estructura clara basada en roles, relaciones documentales, tareas, notificaciones y comunicacion interna.

Desde un punto de vista tecnico, el proyecto permite defender varios aspectos relevantes:

- separacion razonable de responsabilidades entre controladores, acciones, modelos y vistas;
- control de acceso por autenticacion, rol y reglas de negocio;
- trazabilidad documental mediante variantes y estados;
- existencia de pruebas automatizadas ejecutables;
- soporte combinado para interfaz web y API protegida.

El sistema no se limita a ser una maqueta visual. Existe logica real de negocio en la gestion de PR, documentos, plantillas, variantes, tareas y chat, lo que permite justificar su utilidad y su complejidad tecnica.

## 2. Aportaciones del proyecto

- unificacion de la informacion en una sola herramienta;
- mejora de la coordinacion entre gestor, docente y revisor;
- visibilidad de trabajo pendiente mediante tareas y notificaciones;
- reduccion de errores en estados documentales gracias a reglas de exclusividad;
- refuerzo de la calidad mediante pruebas y revisiones de accesibilidad.

## 3. Limitaciones actuales

- la aplicacion esta pensada sobre todo para entorno local o de demostracion controlada;
- no existe integracion completa con sistemas institucionales externos;
- faltan metricas avanzadas y cuadros de mando;
- la parte documental puede seguir creciendo con historicos y auditorias mas detalladas;
- la experiencia movil puede seguir refinandose.

## 4. Mejoras futuras

Las mejoras futuras mas razonables serian:

- incorporar un panel estadistico para gestor;
- ampliar el sistema de auditoria y actividad del usuario;
- integrar almacenamiento documental externo real;
- añadir exportacion de informes;
- ampliar pruebas API y cobertura de frontend;
- introducir filtros avanzados en tareas, documentos y notificaciones;
- mejorar la gestion de adjuntos y comentarios de revision.

## 5. Valor academico

El proyecto permite demostrar competencias en:

- analisis de requisitos;
- modelado de datos;
- desarrollo backend con framework moderno;
- diseño de interfaz web;
- seguridad y control de acceso;
- pruebas de software;
- documentacion tecnica y presentacion.

## 6. Cierre

En conclusion, el trabajo realizado da como resultado una aplicacion funcional, coherente y defendible para un proyecto de fin de ciclo. El estado actual del sistema permite presentar una solucion completa, con base tecnica suficiente y con un recorrido claro de evolucion futura.
